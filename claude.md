## 1. 项目目标

构建一个迷你的"五险一金"计算器Web应用，核心功能是：
- 根据预设的员工工资数据和城市社保标准
- 计算公司为每位员工应缴纳的社保公积金费用
- 将结果清晰地展示出来

## 4. 核心业务逻辑

### 4.1 计算流程

```
1. 从 salaries 表读取所有数据
   ↓
2. 按员工姓名 (employee_name) 分组，计算"年度月平均工资"
   ↓
3. 从 cities 表获取指定城市的标准（year、base_min、base_max、rate）
   ↓
4. 对每位员工，将"年度月平均工资"与基数上下限比较，确定"最终缴费基数"
   - 低于下限 → 使用下限
   - 高于上限 → 使用上限
   - 在区间内 → 使用平均工资本身
   ↓
5. 根据"最终缴费基数" × rate = "公司应缴纳金额"
   ↓
6. 将结果存入 results 表（每次计算前清空旧数据）
```

### 4.2 计算示例

以2024年佛山标准为例（rate=0.14, base_min=4546, base_max=26421）：

| 员工 | 月平均工资 | 缴费基数 | 计算逻辑 | 公司缴纳金额 |
|------|-----------|---------|---------|-------------|
| 员工1 | 30000 | 26421 | 超上限用上限 | 26421 × 0.14 = 3698.94 |
| 员工2 | 15000 | 15000 | 区间内用实际 | 15000 × 0.14 = 2100.00 |
| 员工3 | 4000 | 4546 | 低于限用下限 | 4546 × 0.14 = 636.44 |

## 5. 前端页面设计

### 5.1 `/` 主页（导航入口）

**定位**：应用入口页面和导航中枢

**布局**：
```
┌─────────────────────────────────────────┐
│           五险一金计算器                 │
├──────────────────┬──────────────────────┤
│   ┌──────────┐   │   ┌──────────┐       │
│   │数据上传  │   │   │结果查询  │       │
│   │  [卡片]  │   │   │  [卡片]  │       │
│   └──────────┘   │   └──────────┘       │
└──────────────────┴──────────────────────┘
```

**交互**：
- 点击"数据上传"卡片 → 跳转到 `/upload`
- 点击"结果查询"卡片 → 跳转到 `/results`

### 5.2 `/upload` 数据上传与操作页

**定位**：后台操作控制面板

**功能**：

1. **上传数据按钮**
   - 点击后弹出文件选择器
   - 分别选择 `cities.xlsx` 和 `salaries.xlsx`
   - 验证文件格式和列名
   - 追加插入到 Supabase 对应表中

2. **年份选择下拉框**
   - 从 cities 表获取可用年份
   - 用户选择要计算的年份

3. **执行计算按钮**
   - 触发核心计算逻辑
   - 清空 results 表旧数据
   - 存入新计算结果

### 5.3 `/results` 结果查询与展示页

**定位**：计算成果展示页面

**功能**：
- 页面加载时自动从 results 表获取所有数据
- 使用 Tailwind CSS 表格展示结果

**表格样式**：
```
┌─────────────────────────────────────────────────────────┐
│                    计算结果                              │
├──────┬──────────┬──────────┬──────────┬──────────────┤
│ 序号 │ 员工姓名 │ 平均工资 │ 缴费基数 │ 公司缴纳金额 │
├──────┼──────────┼──────────┼──────────┼──────────────┤
│  1   │  张三    │ 30000.00 │ 26421.00 │   3698.94    │
│  2   │  李四    │ 15000.00 │ 15000.00 │   2100.00    │
│  3   │  王五    │  4000.00 │  4546.00 │    636.44    │
└──────┴──────────┴──────────┴──────────┴──────────────┘
```

```bash
# Supabase
NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
```

## 9. 关键注意事项

1. **Excel 列名映射**：cities.xlsx 原文件中 `city_namte` 需映射为 `city_name`
2. **数据上传策略**：追加插入，不覆盖已有数据
3. **计算结果策略**：每次计算前清空 results 表
4. **年份选择**：用户需选择年份进行计算
5. **可扩展性**：cities 表设计支持多城市，当前只需佛山数据
